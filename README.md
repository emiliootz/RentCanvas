# RentCanvas

A private tenant portal and rent management system for a single landlord managing one property with multiple units.

## Tech Stack

- **Next.js 16** (App Router, TypeScript)
- **Clerk** – Authentication (email/password, invite-based onboarding)
- **Stripe** – ACH bank payments, invoicing
- **PostgreSQL** – Primary database
- **Prisma** – ORM + migrations
- **Bootstrap 5** – UI styling

## Setup Instructions

### Prerequisites

- Node.js 18+
- PostgreSQL running locally (or a hosted instance)
- A [Clerk](https://clerk.com) account
- A [Stripe](https://stripe.com) account

---

### 1. Clone and install dependencies

```bash
git clone <repo-url>
cd RentCanvas
npm install
```

### 2. Configure environment variables

```bash
cp .env.example .env.local
```

Edit `.env.local` with your values:

| Variable | Where to find it |
|---|---|
| `DATABASE_URL` | Your PostgreSQL connection string |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk Dashboard → API Keys |
| `CLERK_SECRET_KEY` | Clerk Dashboard → API Keys |
| `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` | Stripe Dashboard → Developers → API Keys |
| `STRIPE_SECRET_KEY` | Stripe Dashboard → Developers → API Keys |
| `STRIPE_WEBHOOK_SECRET` | Generated by `stripe listen` (see below) |
| `NEXT_PUBLIC_APP_URL` | `http://localhost:3000` for local dev |
| `ADMIN_CLERK_USER_ID` | Your Clerk user ID (found in Clerk Dashboard after signing up) |

### 3. Configure Clerk

1. Go to [Clerk Dashboard](https://dashboard.clerk.com)
2. Create a new application
3. Under **User & Authentication → Email, Phone, Username**:
   - Enable **Email address** (required)
   - Enable **Password** (required)
   - Disable Google, Apple, and all social logins
4. Under **Email & SMS → Templates**, confirm email verification is on
5. Under **Restrictions**, set to "Allowlist only" or keep invite-only to prevent public sign-ups
6. Copy your API keys into `.env.local`

### 4. Run database migrations

```bash
npx prisma migrate dev --name init
```

This creates all tables in your PostgreSQL database.

### 5. Seed sample data

```bash
npm run db:seed
```

This creates:
- 1 property (Maple Street Apartments)
- 3 units (1A, 2B, 3C)
- 3 sample tenants (Alice, Bob, Carol) with placeholder Clerk IDs
- Leases and lease participants
- Sample invoices in various states

> **Note:** The seed uses placeholder Clerk IDs. After you create real accounts, update the `clerkId` values via the admin UI or Prisma Studio.

### 6. Start the development server

```bash
npm run dev
```

Visit [http://localhost:3000](http://localhost:3000).

### 7. Set up Stripe webhooks (Phase 3)

```bash
# Install Stripe CLI
brew install stripe/stripe-cli/stripe

# Log in
stripe login

# Forward webhooks to your local server
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

Copy the `whsec_...` secret into `STRIPE_WEBHOOK_SECRET` in `.env.local`.

## User Roles

| Role | Routes | Access |
|------|--------|--------|
| `admin` | `/admin/*` | All records, all tenants |
| `tenant` | `/dashboard/*` | Their own invoices, payments, receipts, maintenance, documents |

The admin is the landlord. There is one admin account for this MVP.

---

## Onboarding a Tenant

1. Admin invites the tenant by email via the Clerk dashboard (or via the in-app invite flow in Phase 2)
2. Tenant receives an email with an invite link
3. Tenant sets a password and verifies their email
4. Admin links the tenant's Clerk ID to their `TenantProfile` in the app
5. Tenant logs in and sees only their own data

---

## Payment Flow (Phase 3)

1. Admin generates monthly invoices for all active tenants
2. Tenant sees their invoice on the dashboard
3. Tenant connects a US bank account via Stripe (ACH)
4. Tenant pays the full or partial invoice amount
5. Stripe processes ACH and sends a webhook
6. App updates invoice status, creates a Receipt, and emails the tenant
